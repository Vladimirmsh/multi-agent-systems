# 🎧 Multi-Agent Customer Support System

Цей проєкт — інтелектуальна мультиагентна система для обробки клієнтських запитів. Вона автоматично класифікує звернення, надає відповіді на основі бази знань (RAG) або пошуку в інтернеті, та вміє ескалювати критичні запити на живих операторів.

## 🏗 Архітектура (Routing Pattern)

Система побудована за допомогою **LangGraph**.
1. **Router Agent:** Приймає запит і за допомогою `structured_output` класифікує його (категорія та терміновість).
2. **Docs Agent:** Якщо запит стосується продукту (`product`), використовується RAG для пошуку у внутрішній базі знань.
3. **Web Search Agent:** Якщо запит загальний (`general`), агент використовує DuckDuckGo для пошуку зовнішньої інформації (напр., інтеграції).
4. **Escalation Agent:** Якщо Router визначив проблему як критичну, або якщо Docs/Web агенти мають рівень впевненості (confidence) нижче 0.75, запит передається людині через месенджер із збереженням логу.

🧪 Тестування (LLM-as-a-Judge)
У проєкті реалізовано автоматизовані тести, де окрема LLM виступає як суддя і перевіряє коректність маршрутизації та якості відповідей:

Bash
pytest test_evals.py -v


Для тестування бота можна вкиористати запити: 

1. Тест Docs Agent (Категорія product, пошук по базі знань)
"Доброго дня! Розкажіть, будь ласка, які у вас є тарифи на підписку і чи можу я повернути кошти протягом першого тижня, якщо сервіс мені не підійде?"
Що перевіряється: Router повинен класифікувати це як product і направити до Docs Agent. Агент має витягнути дані з твого RAG (або mock-функції) про тарифи та політику повернень і сформувати ввічливу відповідь.
2. Тест Web Search Agent (Категорія general, пошук в інтернеті)
"Підкажіть, чи можу я інтегрувати вашу платформу з Google Sheets або Zapier для автоматизації звітів? Як саме це працює?"
Що перевіряється: У твоїй внутрішній базі знань (швидше за все) не буде детальних інструкцій про всі зовнішні сервіси світу. Router має визначити це як general запит. Завдання перейде до Web Search Agent, який через DuckDuckGo знайде актуальну інформацію про інтеграції з цими сервісами.
3. Тест Escalation Agent (Категорія critical, термінова ескалація)
"Ваша система лежить вже дві години!! Я втрачаю реальних клієнтів і гроші! Негайно поверніть кошти за цей місяць і зв'яжіть мене з керівництвом!"
Що перевіряється: Router має розпізнати високу терміновість (urgency: critical) або віднести це до категорії скарг/критичних помилок. Запит має оминути пошукові агенти і відразу потрапити до Escalation Agent. Система повинна зберегти JSON-звіт у папку escalations, залогувати "відправку" в Slack і повернути користувачу повідомлення, що його запит передано живому оператору.
