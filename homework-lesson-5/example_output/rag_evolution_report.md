# Звіт: Еволюція методів RAG (від базового до сучасних підходів)

Цей звіт порівнює класичну архітектуру RAG, описану у внутрішній базі знань, із сучасними методами, що активно розвиваються в індустрії станом на 2024-2025 роки.

## 1. Класичний RAG (згідно з базою знань)
Класичний підхід (Naive RAG) базується на простій схемі:
*   **Індексація:** Документи перетворюються на векторні ембеддінги.
*   **Пошук (Retrieval):** За запитом користувача система шукає найбільш схожі за вектором фрагменти тексту.
*   **Генерація:** Знайдені фрагменти додаються до контексту LLM для формування відповіді.

**Обмеження:** Часто страждає від низької точності при складних запитах, не враховує зв'язки між різними частинами документів та може "загубити" контекст у великих обсягах даних.

## 2. Сучасні та просунуті методи RAG (Advanced RAG)
Останні дослідження (2024-2025) фокусуються на подоланні обмежень класичного підходу:

### А. GraphRAG (Графовий RAG)
Це один із найперспективніших напрямків. Замість простого пошуку за схожістю, дані структурують у вигляді **графа знань (Knowledge Graph)**.
*   **Перевага:** Дозволяє моделі розуміти складні зв'язки між сутностями (наприклад, "як цей документ пов'язаний з іншим через спільну тему").
*   **Актуальність:** Використовується для глобального аналізу даних, де потрібно синтезувати інформацію з багатьох джерел одночасно.

### Б. Self-RAG та Corrective RAG (CRAG)
Це методи "саморефлексії" моделі:
*   **Self-RAG:** Модель сама оцінює, чи потрібен їй пошук, чи достатньо власних знань, а також перевіряє релевантність знайденої інформації.
*   **Corrective RAG:** Система автоматично перевіряє якість знайдених документів. Якщо вони нерелевантні, система може ініціювати пошук в інших джерелах (наприклад, у вебі).

### В. Гібридний пошук (Hybrid Search)
Поєднання векторного пошуку (семантичного) з класичним ключовим пошуком (BM25). Це дозволяє краще знаходити специфічні терміни, назви або артикули, де векторний пошук може помилятися.

## 3. Порівняльна таблиця

| Характеристика | Класичний RAG | Advanced RAG (GraphRAG, Self-RAG) |
| :--- | :--- | :--- |
| **Метод пошуку** | Векторна схожість | Графи знань, семантика + ключові слова |
| **Якість контексту** | Базова | Висока (враховує зв'язки та релевантність) |
| **Складність впровадження** | Низька | Висока |
| **Надійність** | Середня | Висока (завдяки самоперевірці) |

## Висновок
Хоча класичний RAG залишається фундаментом, сучасні системи переходять до **Advanced RAG**. Використання графів знань (GraphRAG) та механізмів самокорекції (Self-RAG) дозволяє створювати набагато розумніші системи, які здатні не просто "знаходити текст", а "розуміти контекст та зв'язки" у великих масивах даних.
