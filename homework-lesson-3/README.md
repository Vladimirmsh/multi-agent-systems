# AI Research Agent

Автономний агент-дослідник, побудований на базі **LangChain (LangGraph)** та найновішої моделі **Gemini 3.1 Flash Lite**. 
Агент вміє самостійно гуглити інформацію, читати статті, аналізувати їх та зберігати результати у вигляді структурованих Markdown-звітів.

## 🚀 Можливості
- **Автономний пошук:** Використовує DuckDuckGo (`web_search` через бібліотеку `ddgs`).
- **Читання сторінок:** Витягує чистий текст із сайтів за допомогою Trafilatura (`read_url`).
- **Context Engineering:** Автоматично обрізає завеликі тексти, щоб не переповнити контекстне вікно LLM.
- **Пам'ять:** Підтримує зв'язний діалог у межах сесії (`MemorySaver`).
- **Візуалізація процесу:** Відображення кроків агента (викликів інструментів) у терміналі в реальному часі.
- **Збереження файлів:** Самостійно створює файли звітів у папці `output/`.

📄 Приклад результату (example_output/report.md)
Ось приклад того, як виглядає згенерований звіт:

Порівняння підходів до побудови RAG (2024)
1. Naive RAG
Найпростіший підхід. Документи розбиваються на чанки однакового розміру, векторизуються і зберігаються в базі.

Плюси: Легко і швидко імплементувати.
Мінуси: Втрата контексту на межі чанків, низька точність при складних запитах.
2. Sentence-Window Retrieval
Індексуються окремі речення. При пошуку знаходиться найбільш релевантне речення, але в LLM передається розширене "вікно" (кілька речень до і після).

Плюси: Висока точність пошуку (бо шукаємо конкретний факт) + збереження контексту для LLM.
Мінуси: Більше навантаження на векторну базу.
3. Parent-Child Retrieval (Auto-merging)
Документ розбивається на ієрархію (Parent -> Child). Векторизуються дрібні Child-чанки. Якщо багато Child-чанків з одного Parent-чанка виявляються релевантними, в LLM передається весь Parent-чанк.

Плюси: Ідеальний баланс між точністю пошуку та повнотою контексту.
Мінуси: Складна архітектура індексування.

### 💡 Маленька порада наостанок:
Оскільки ми виправляли помилку з бібліотекою пошуку вручну через термінал, переконайтеся, що у вашому файлі **`requirements.txt`** написано `ddgs>=1.0.0` (або просто `ddgs`), а **НЕ** `duckduckgo-search`. Це гарантує, що якщо ви будете запускати проєкт на іншому комп'ютері, він встановить правильну версію бібліотеки!
